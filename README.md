# Wardrobe Web App

A modern, mobile-first wardrobe management application built with a NestJS backend and a React (TSX) frontend. The project provides comprehensive clothing item management, a dynamic outfit builder, AI-powered auto-tagging, and a virtual fitting room.

## 🚀 Key Features

- **Clothing Management:** Full CRUD operations for detailed wardrobe items categorized by types, colors, and hierarchical storage locations.
- **AI Auto-Tagging:** Built-in integration with Google Gemini to automatically detect clothing attributes (category, color, style) from images upon upload.
- **Background Processing & Webhooks:** High-performance background image processing (background removal) utilizing Cloudinary's infrastructure, receiving updates securely via Webhook and notifying clients in real-time through WebSockets.
- **Outfit Builder & Virtual Fitting Room:** An intuitive Drag-and-Drop (D&D) interface to combine multiple clothing items into complete outfits, utilizing Cloudinary AI-processed images for compositing.
- **Bulk Import/Export:** Functionality to bulk import clothing inventory via Excel templates, and export existing catalogs for backup.
- **Authentication & Security:** Robust Auth supporting JWT session management, as well as Single Sign-On (SSO) with Google and Facebook via Passport.js.
- **Comprehensive Taxonomies:** Dynamic settings for managing custom item attributes, properties, and multi-level physical storage location hierarchies.

## 🛠️ Tech Stack & Architecture

**Frontend (React/Vite):**
- **Library:** React (Functional Components & Hooks)
- **Styling:** Tailwind CSS
- **Typing:** Strict TypeScript 
- **API Client:** Autogenerated API client using Orval
- **Deployment:** Vercel (CD)

**Backend (NestJS):**
- **Framework:** NestJS (Node.js)
- **Database:** MongoDB via Mongoose
- **Validation & Docs:** `class-validator`, Swagger API
- **AI/ML:** Google Gemini 
- **Third-party Services:** Cloudinary (Storage, Transformations, AI Background Removal), OAuth (Google, Facebook)
- **Deployment:** Dokploy (CD)

## 🐳 Running Locally

To simplify the local setup process, the stack can be launched via Docker Compose and NPM scripts.

1. **Clone the repository:**
   ```bash
   git clone <repository_url>
   cd wardrobe-webapp
   ```

2. **Configure Environment Variables:**
   Create `.env` files in both the `back-end` and `front-end` based on example files:
   ```env
   # Backend example
   PORT=3000
   MONGO_URI=mongodb://localhost:27017/wardrobe  
   JWT_SECRET=your_jwt_secret_key
   CLOUDINARY_CLOUD_NAME=your_cloud_name
   CLOUDINARY_API_KEY=key
   CLOUDINARY_API_SECRET=secret
   GEMINI_API_KEY=key
   GOOGLE_CLIENT_ID=oauth_id
   ```

3. **Running the services locally:**
   Start the backend via Docker:
   ```bash
   docker-compose up --build
   ```
   Start the frontend natively:
   ```bash
   cd front-end
   npm install
   npm run dev
   ```

## 📜 CI/CD Workflows

Currently, Continuous Deployment (CD) relies on robust push-to-main setups:
- **Dokploy:** Watches the `main` branch to rebuild and continuously deploy the NestJS Backend Docker container.
- **Vercel:** Watches the `main` branch to securely host, build, and distribute the React Frontend automatically.
