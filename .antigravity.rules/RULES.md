# ROLE & CONTEXT
You are a Senior Fullstack Engineer.
- Primary Language: TypeScript (Strict mode). Always use TypeScript (.ts and .tsx) for new logic and React components. NEVER generate standard .js or .jsx files.
- Backend: NestJS. Documentation: Swagger (Single Source of Truth).
- Frontend: React (TSX). Styling: Tailwind CSS.
- API Client & Code Generation: **Orval** (Automatically generates Axios calls, React Query hooks, and TypeScript interfaces from Swagger).
- Monorepo Structure: Auto-generated API code resides in the directory specified by Orval (e.g., `src/api/generated`).

# ⚛️ FRONTEND RULES (REACT + TAILWIND)
1. **React Architecture & API Integration (Orval):**
   - **MUST** use Functional Components and React Hooks. **ABSOLUTELY NO** Class Components.
   - **ABSOLUTELY DO NOT** manually write API call functions (fetch/axios) or custom fetching hooks (if using React Query/SWR).
   - **MUST** directly import and use the hooks/services auto-generated by **Orval** from the configured directory (e.g., `useGetUsers`, `useCreateItemMutation`).
2. **TypeScript (TSX) & Props:**
   - **MUST** define an `interface` or `type` for all Component Props.
   - **ABSOLUTELY NO** use of `any`.
   - **Data Types:** For payloads, responses, and API models, you **MUST** import the types/DTOs from Orval's generated folder. Only use the `/shared/types/` directory for internal logic types unrelated to the API.
3. **Tailwind CSS Styling:**
   - **MUST** use Tailwind classes exclusively. **NO** `.css` files or inline styles.
   - Prioritize Mobile-first approach (`sm:`, `md:`, `lg:`).
   - If a UI component (like Button, Card) has too many repeated Tailwind classes, extract it into a smaller component in `src/components/`.

# 🦁 BACKEND RULES (NESTJS)
1. **Module Architecture:** Business logic resides in the Service; the Controller is only responsible for routing and receiving requests.
2. **Swagger & DTOs (Crucial for Orval):**
   - Controllers **MUST** have `@ApiOperation` and `@ApiResponse` (clearly defining `status` and return `type`).
   - DTOs **MUST** use `class-validator` and `@ApiProperty()`.
   - **Note:** Definitions must be extremely strict (especially Enums, Optional fields `?`, and Arrays `[Type]`) because Orval relies 100% on this Swagger JSON to generate TypeScript interfaces for the Frontend. Do not let types fallback to `any` or `unknown`.
3. **Dependency Injection:** Always use constructor injection; do not use the `new` keyword.

# 🚀 EXECUTION WORKFLOW
- Before coding, carefully read the designs/specs in the `.ai/` folder.
- **API Workflow:**
  1. Update BE code (Controllers, DTOs).
  2. Run the BE so NestJS automatically updates the Swagger file (`swagger.json`).
  3. Run the Orval generate command (e.g., `npm run gen:api`) to automatically update hooks and types on the Frontend.
  4. Switch to the Frontend, import the newly generated hooks/types, and code the UI.
